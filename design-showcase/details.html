1. Color Palette & Visual Identity

The neon color scheme that defines the game's mood and readability.

Cyberpunk Neon

Hot pink, cyan, and purple on dark backgrounds. High contrast, aggressive energy. Think Hotline Miami or Cyberpunk aesthetics.

PROS

  • Extremely eye-catching
  • Strong brand identity
  • Great for glow/bloom effects
  • Trendy, appeals to wide audience

CONS

  • Can cause eye strain in long sessions
  • Hard to differentiate many element types
  • Oversaturated market with this look

Midnight Aurora

Deep blues and teals with green/gold accents. Cooler, more refined palette. Calming yet vibrant. Think Northern Lights.

PROS

  • Easy on the eyes for long play
  • Elegant, premium feel
  • Good readability contrast
  • Distinct from typical neon games

CONS

  • Less immediately striking
  • Fewer "wow" moments visually
  • Can feel muted for action games

Synthwave Sunset

Orange, magenta, and violet gradients. Warm retro-futurism. Think Outrun, sunset grids, and 80s sci-fi.

PROS

  • Warm, inviting aesthetic
  • Strong nostalgia appeal
  • Great gradient opportunities
  • Unique for tower defense genre

CONS

  • Limited cool-tone options
  • Warm colors harder for "danger" signals
  • Synthwave fatigue in indie scene

Monochrome + Accent

Mostly white/gray on black with a single accent color per element type. Ultra-minimal. Think Superhot or Return of the Obra Dinn.

PROS

  • Maximum readability
  • Color = meaning (each type = one hue)
  • Strikingly minimal and unique
  • Easiest to maintain visual consistency

CONS

  • Can feel stark or cold
  • Less spectacle for VFX moments
  • Some players find it boring
  • Screenshots may look plain

2. Enemy Design Philosophy

How enemies are differentiated — what makes each wave feel different?

Shape = Type

Enemy shape directly communicates behavior. Circles are fast, squares are tanky, triangles deal damage. Instantly readable with no learning curve.

PROS

  • Zero learning curve — shape IS info
  • Perfect for abstract theme
  • Scales to many enemy types (3-sided to N-sided)
  • Easy to differentiate at a glance

CONS

  • Limited to ~8-10 distinct shapes
  • Harder to show "elite" variants
  • No personality or character

Color = Resistance

Enemy color shows what they resist. Red enemies resist fire, blue resist ice, etc. Shape shows behavior, color shows weakness. Dual-encoding.

PROS

  • Two info channels (shape + color)
  • Natural tower/enemy counter system
  • More strategic depth
  • Colorful, visually dynamic waves

CONS

  • Accessibility concern (colorblindness)
  • More complex to learn
  • Can clash with palette choices
  • Needs clear legend/tutorial

Composite / Modular

Enemies are built from parts: a core shape + orbiting shields, rings, or layers. Strip layers to kill. Bosses are complex multi-part entities.

PROS

  • Visual damage feedback (layers break)
  • Infinite enemy variety from combinations
  • Bosses feel epic and multi-phase
  • Unique to this game

CONS

  • More complex rendering
  • Harder to read at a glance
  • Balance complexity increases
  • Performance cost with many enemies

3. In-Match Economy

How players earn and spend resources during a match.

Simple Gold

One resource: gold. Earn per kill + wave bonus. Spend on towers and upgrades. Sell towers for partial refund. Classic and clean.

PROS

  • Easiest to understand
  • Every decision is gold vs gold
  • Proven model in every major TD
  • Simple UI — one number

CONS

  • Less strategic depth
  • No tension between resource types
  • Optimal play can be "solved"

Gold + Interest / Banking

Gold with an interest system: unspent gold earns bonus gold each wave. Creates risk/reward tension — save for interest or spend for defense?

PROS

  • Deep economic strategy layer
  • Risk/reward each wave
  • Rewards skilled players who can greed
  • Popular in Bloons, autobattlers

CONS

  • Harder to teach new players
  • "Greeding" can feel unintuitive
  • Snowball potential (rich get richer)

Dual Resources (Energy + Matter)

Two resources: Energy (from kills, powers abilities) and Matter (per wave, builds towers). Different towers cost different ratios. Fits sci-fi/abstract theme.

PROS

  • More build diversity (budget constraints)
  • Natural tower specialization
  • Thematic for abstract setting
  • Prevents one-strategy dominance

CONS

  • More complex UI (two counters)
  • Harder to balance
  • Can feel frustrating if one resource starved
  • Longer learning curve

4. Roguelite Loop Structure

What persists between runs and how the meta-progression works.

Unlock Tree

Earn "shards" each run. Spend on a skill tree that permanently unlocks new towers, upgrades, and modifiers. Linear progression with branches.

PROS

  • Clear progression path
  • Players always working toward something
  • Easy to gate content
  • Visual satisfaction (filling out a tree)

CONS

  • Can feel like a grind wall
  • Early runs may feel incomplete
  • Finite — once fully unlocked, loop ends

Deck Building

Start each run by drafting a "deck" of available towers from your collection. Between runs, earn new tower cards to expand your collection. Each run uses a subset.

PROS

  • Every run feels strategically different
  • Collection-building is addictive
  • Natural synergy discovery
  • Fits the "12-15 towers" roster perfectly

CONS

  • More complex pre-run UI needed
  • Bad drafts can feel unwinnable
  • Need enough towers for varied decks

Wave Rewards (Slay the Spire style)

After every N waves, pick 1 of 3 random rewards: new tower, upgrade, passive ability, or relic. Build your loadout as you go. Between runs, unlock new items for the reward pool.

PROS

  • Proven, beloved model (StS, Hades)
  • Moment-to-moment excitement
  • Adapting build mid-run is thrilling
  • Huge variety in runs

CONS

  • RNG can frustrate (bad offers)
  • Need large item/relic pool
  • Balance is the hardest of the three

5. UI / HUD Layout

Where information lives on screen during gameplay.

Bottom Bar

Tower selection and info in a bar at the bottom. Resources and wave info at the top. Maximum vertical play space. Mobile-friendly thumb zone.

PROS

  • Familiar layout (most TD games)
  • Thumb-friendly for mobile/touch
  • Towers always visible
  • Good for widescreen monitors

CONS

  • Reduces vertical play space
  • Can feel cramped on small screens
  • Info at top + bottom splits attention

Right Side Panel

Collapsible panel on the right with towers, upgrades, and info. Top bar for resources. Play area takes most of the screen.

PROS

  • All info consolidated on one side
  • Room for detailed tower info
  • Can show upgrade paths inline
  • Collapsible for full-screen play

CONS

  • Reduces horizontal play space
  • Less mobile-friendly
  • Can feel "PC app"-like

Minimal + Radial Menu

Almost no permanent HUD. Right-click/hold on grid to open a radial tower menu. Info appears contextually. Maximum immersion.

PROS

  • Maximum play area — full immersion
  • Clean, modern feel
  • Fits abstract/minimal aesthetic perfectly
  • Info only when you need it

CONS

  • Slower tower selection (extra click)
  • Harder for new players to discover
  • Can't see all options at once
  • Radial menus are tricky on mobile

6. Tech Stack

The engine/framework powering the game. Affects dev speed, performance, and capabilities.

Phaser 3

Full-featured 2D game framework. Built-in physics, scenes, input, tilemaps, tweens, particles. Huge community. WebGL with Canvas fallback.

PROS

  • Batteries-included (everything built in)
  • Massive community and tutorials
  • Scene management out of the box
  • Good TypeScript support
  • Mature and battle-tested

CONS

  • Large bundle size (~1MB)
  • Opinionated architecture
  • Some abstractions add overhead
  • Custom rendering harder

PixiJS + Custom

Fast WebGL renderer only — you build game logic yourself. Maximum control over rendering. Add libraries as needed (Howler for audio, etc).

PROS

  • Fastest 2D WebGL renderer
  • Total control over architecture
  • Smaller base bundle
  • Best for custom visual effects
  • No framework opinions to fight

CONS

  • Must build game systems yourself
  • No built-in scene management
  • More boilerplate code needed
  • Steeper learning curve initially

Raw Canvas/WebGL

No framework — pure Canvas 2D or raw WebGL. Zero dependencies. Maximum performance ceiling. You own every line of rendering code.

PROS

  • Zero dependencies — smallest possible
  • Full understanding of every system
  • Maximum performance potential
  • Educational value

CONS

  • Reinventing the wheel constantly
  • Slowest development speed
  • Must build sprite batching, etc
  • No community patterns to follow
  • High risk of technical debt
